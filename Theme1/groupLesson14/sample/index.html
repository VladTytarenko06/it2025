O(1)

Константний час

Швидкий доступ до елементу, хеш-таблиці, робота зі стеком/чергою без обходу.

O(log n)

Логарифм

Двійковий пошук, балансовані дерева, роботи по діленню проблеми навпіл.

O(n)

Лінійно

Повний обхід даних, перевірка цілісності записів, лінійний пошук.

O(n log n)

Сортування бойового рівня

Quicksort/mergesort/heap, розумний компроміс між швидкістю та ресурсами.

O(n²)

Дорого

Прості сортування (bubble/selection) або повні порівняння всіх з усіма.

Як читати результати

  • Кількість операцій відображає теоретичну складність: слідкуйте за різницею між O(n²) та O(n log n).
  • Час залежить від вашого пристрою, але тренд співпадає з теорією.
  • Кроки показують послідовність вибору, поділу, перестановок та обходу.
  • Щільність перешкод у графі демонструє, як зростає навантаження на пошук шляху.
Локальна симуляція
selectionSort ➜ O(n²)
quickSort      ➜ O(n log n)
linearSearch   ➜ O(n)
binarySearch   ➜ O(log n)
BFS / DFS      ➜ O(V + E)

Можна змінювати розмір даних та густоту перешкод, щоб демонструвати стрибок у витратах.
