## Творче завдання: Прототип ІАС для магістерського дослідження

### Етап I: Проєктування (Логіка системи)

**1. Визначення домену**
Оберіть 4 ключові об'єкти вашої магістерської роботи (наприклад: _Сенсори, Інциденти, Ресурси, Оператори_). Опишіть їх у текстовому файлі `concept.txt`.

> **Контрольний стан:** У папці проекту є файл `concept.txt` з переліком об'єктів та їхніх 3-4 атрибутів.

**2. Візуалізація структури (PlantUML)**
Створіть файл `model.puml`. Опишіть зв'язки між об'єктами (наприклад, "Один Оператор керує багатьма Сенсорами").

> **Контрольний стан:** При натисканні `Alt+D` у VS Code (з плагіном PlantUML) відображається коректна схема з вузлами та лініями зв'язку.

**3. Протокол операції**
Опишіть у `workflow.md` послідовність дій: що стається в системі, коли виникає подія (наприклад, "Виявлено перевищення показника -> Створення інциденту -> Призначення екіпажу").

> **Контрольний стан:** Файл `workflow.md` містить нумерований список кроків "життєвого циклу" даних.

---

### Етап II: База даних (SQL)

**4. Створення таблиць (DDL)**
Створіть файл `schema.sql`. Напишіть команди `CREATE TABLE` для ваших 4-х об'єктів. Використовуйте `INTEGER PRIMARY KEY` для ID.

> **Контрольний стан:** Після виконання скрипту у VS Code через розширення _SQLite Viewer_ видно файл `.db` з порожніми таблицями.

**5. Генерація тестового середовища**
Створіть файл `data.sql` з командами `INSERT`. Додайте по 5-10 записів. Використовуйте абстрактні назви: "Об'єкт*А", "Тип_1", "Статус*ОК".

> **Контрольний стан:** Таблиці в _SQLite Viewer_ заповнені даними, немає помилок "Constraint failed".

**6. Аналітичний вибір (Queries)**
Створіть файл `analysis.sql`. Напишіть 3 запити: один на фільтрацію (`WHERE`), один на підрахунок (`COUNT`) і один на об'єднання двох таблиць (`JOIN`).

> **Контрольний стан:** Виконання запиту в консолі VS Code повертає таблицю з очікуваним результатом (наприклад, список лише "активних" об'єктів).

---

### Етап III: Бекенд (Python)

**7. Конектор до даних**
Створіть `app.py`. Напишіть код, що підключається до `database.db` за допомогою бібліотеки `sqlite3` і виводить версію бази даних.

> **Контрольний стан:** В терміналі після команди `python app.py` з’являється повідомлення про успішне підключення.

**8. Функція пошуку**
Додайте в `app.py` функцію, яка приймає текстовий запит і шукає його в одній з таблиць (наприклад, пошук техніки за назвою).

> **Контрольний стан:** Виклик функції з коду повертає список знайдених рядків у термінал.

**9. Логічний фільтр «Тривога»**
Напишіть функцію, яка перевіряє певний числовий показник (наприклад, "час очікування" або "рівень ризику") і повертає статус "WARNING", якщо значення перевищує норму.

> **Контрольний стан:** Скрипт виводить у термінал різні статуси залежно від вхідних даних з вашої таблиці.

---

### Етап IV: Інтерфейс та Звітність

**10. Створення веб-каркасу (Streamlit)**
Імпортуйте `streamlit` у файл `dashboard.py`. Напишіть `st.title("Ваша Назва ІАС")`.

> **Контрольний стан:** Після команди `streamlit run dashboard.py` у браузері відкривається порожня сторінка з вашим заголовком.

**11. Інтерактивна таблиця та графік**
Додайте на сторінку відображення вашої SQL-таблиці (`st.dataframe`) та одну просту діаграму (`st.bar_chart`), що показує розподіл ресурсів чи статусів.

> **Контрольний стан:** На веб-сторінці видно дані з вашої бази у вигляді графіків, які змінюються при оновленні бази.

**12. Презентаційний опис для магістерської**
Створіть фінальний `README.md`. Опишіть: яку проблему магістерської вирішує цей прототип, які дані він обробляє та як він допоможе користувачу (командиру/аналітику).

> **Контрольний стан:** Готовий файл `README.md` з чітким поясненням цінності розробки, який можна скопіювати в пояснювальну записку диплома.
